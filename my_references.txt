Install:
	To use 'composer' you have to install it from here:
		https://getcomposer.org/Composer-Setup.exe
	To use 'symfony' you have to install it from here:
		https://get.symfony.com/cli/setup.exe
	To use 'git' you have to install it from here:
		https://github.com/git-for-windows/git/releases/download/v2.32.0.windows.1/Git-2.32.0-64-bit.exe
Create:
	To use symfony to create a controller you need:
		CMD - composer require doctrine/annotations
		CMD - composer require symfony/maker-bundle --dev
		CMD - symfony console make:controller "controller-name"
	to use 'security' install security-bundle
		CMD - composer require security
		CMD - composer require req-check
	or
		Go to https://packagist.org/packages/sensiolabs/security-checker
		And https://github.com/fabpot/local-php-security-checker#local-php-security-checker
		Add in "composer.json" -> scripts -> auto-scripts - '"php-cs":"script"'
		Before you go on "https://github.com/fabpot/local-php-security-checker/releases"
		Download your exe file and then rename it to "php-sc"
		After that you have to add path to enviroment variable path.
	To use templates:
		CMD - composer require twig
	To use profiler (debuger):
		CMD - composer require profiler --dev
		CMD - composer require debug
		CMD - composer unpack symfony/debug

symfony console make:docker:database -> create yaml file for docker container
docker-compose up -d -> create container on docker
docker-compose exec database mysql -u root --password=password -> execute mysql command in container
docker-compose down -> turn down server

		php bin/console make:migration
		php bin/console doctrine:migrations:status
		php bin/console doctrine:migrations:generate
		php bin/console doctrine:migrations:migrate
		php bin/console crud
		php bin/console make:appUser
		php bin/console make:user
Symfony:
	server:
		symfony server (start server)
		symfony server:status (status of server)
		symfony server -d (start in background)
		symfony server:start (start server)
		symfony server:stop (stop server)
